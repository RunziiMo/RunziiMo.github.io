Android开发中的unable to instantiate activity componentinfo问题

发表于 2016-05-14

#Android开发中的unable to instantiate activity componentinfo问题
提前说明：mainfest配置正确，包名也正确。

问题描述如下：新建带Fragment的工程后，显示导包错误如下

import android.support.v7.app.ActionBarActivity; import android.support.v4.app.Fragment;

两个包都出错，将两个jar包添加到buildpath后错误解决。
选择Run As Android Application后却报unable to instantiate activity componentinfo 异常

原因是eclipse解析jar包的时候也会看顺序，只要在Properties–>Java Build Path中将v4和v7两个包的位置提到最上面

之前这个问题也遇到过好几次了，每次都是百度大量资料然后删这删那的解决的。
